对比GET和POST

Posted on 2017-05-27

HTTP

HTTP为“超文本传输协议”，用来保证客户机与服务器之间的通信。

其工作方式是客户机与服务器之间的请求-应答协议。

举例：客户端（浏览器）向服务器提交 HTTP 请求；服务器向客户端返回响应。响应包含关于请求的状态信息以及可能被请求的内容。

两种HTTP请求方法：GET和POST

GET 从指定的资源请求数据

post 向指定的资源提交要被处理的数据

GET与POST的对比

GET POST
后退按钮/刷新 无害 数据会被重新提交（浏览器应该告知用户数据会被重新提交）。
书签 可收藏为书签 不可收藏为书签
编码类型 application/x-www-form-urlencoded application/x-www-form-urlencoded 或 multipart/form-data。为二进制数据使用多重编码。
历史 参数保留在浏览器历史中。 参数不会保存在浏览器历史中。
对数据长度的限制 当发送数据时，GET 方法向 URL 添加数据；URL 的长度是受限制的（URL 的最大长度是 2048 个字符）。 没有限制。
对数据类型的限制 只允许 ASCII 字符。 没有限制。也允许二进制数据。
安全性 与 POST 相比，GET 的安全性较差，因为所发送的数据是 URL 的一部分。 POST 比 GET 更安全，因为参数不会被保存在浏览器历史或 web 服务器日志中。
可见性 数据在 URL 中对所有人都是可见的。 数据不会显示在 URL 中。
缓存 能被缓存 不能缓存

遗传算法

Posted on 2017-05-03

算法介绍

基本思想

遗传算法是一种迭代算法，实质上是一种搜索寻优技术——借助于自然遗传学的选择、复制、交叉、变异等，逐渐演化产生出越来越好的近似解。在每一代，根据问题域中个体的适应度大小，按照优胜劣汰和适者生存的原理，引导搜索过程向最优解逼近，最终产生出代表新的解集的群体，可以作为问题的近似最优解或满意解。

基本概念和术语

1.演化代

算法的迭代步

2.个体

组成种群的每一个解称为一个个体，是遗传算法中用来模拟生物染色体的一定数目的二进制位串，常记为X$_1$(t)，X$_2$(t)，…，等。

3.种群

是由一定数量的个体组成的集合，也是所求解问题的多个解的集合，常记为P(t),t表示其代数，最初的种群即P(0)。

4.种群规模

指种群中包含的个体的数量，常记为N。一般来说，群体规模在整个演化过程中是不变的。

5.染色体

经过编码后得到的代表问题的解的串。每个个体实际上是染色体带有特征的实体。

6.位串

个体的表现形式，一般为二进制串。

7.基因

染色体的每一位称为基因。

8.基因型

染色体性状的内部表现，也即基因组合的模型，或者称为遗传子型。

9.表现型

染色体性状的外部表现。染色体作为遗传物质的主要载体，其基因型决定了个体的表现型。

10.基因模式

指在二进制位串中，某一或某些位置上的具有相似性的个体组成的集合。

11.适应度

个体适应环境的程度（与最优解接近的程度）。

12.平均适应度

指若干个个体的适应度值的算术平均值。

13.父代

根据某种规划来产生其下一代个体称为其下一代个体的父代。

14.子代

根据某种规划来产生的下一代个体称为子代。

15.选择

以一定概率从种群中选出若干个个体的操作。

16.复制

在上一代种群中按照某些指标挑选出若干数量的个体直接插入到新一代种群中去的操作。

17.交叉

指对优选后的父代个体进行基因模式的重组后而产生后代个体的过程，又称为杂交或基因重组。

18.变异

由于各种偶然因素引起染色体的基因改变从而产生新的染色体的过程。

19.遗传算子

包括复制算子、交叉算子、变异算子等。

20.编码

将问题的解通过变换映射到基因空间的方式。

21.解码

编码的逆过程。从基因型到表现型的映射，最后一代种群的最优个体经过解码后，得到的解可以作为问题的近似最优解或满意解。

遗传算法的构成要素

遗传算法应用步骤

1.初始化

随机生成N个个体作为群体P(0)，该种群就是问题可行解的一个集合。设置进化代数计数器归零，设置最大进化代数T。

2.个体评价

将初始种群置于问题的“环境”中，根据适应度函数计算当前群体中各个个体的适应度。

3.终止条件判断

给出终止条件，判断算法是否满足终止条件，若满足则转到(8)

4.选择运算

对初始群体执行选择操作，优良的个体被大量复制，劣质的个体复制的少甚至被淘汰。

5.交叉运算

以交叉概率来进行交叉运算

6.变异运算

以变异概率来进行交叉运算

7.群体P(t)经过选择运算、交叉运算、变异运算之后，得到由N个新个体构成的下一代群体P(t+1)，则转(2)，否则转(4)

8.不断的进化过程中会得到适应度最高的个体，将其作为问题的最优解或满意解输出，终止计算。

编码表示

当用遗传算法求解问题时，把目标问题实际表示转化成染色体位串结构的形式的过程称为编码字符串代表染色体，它是遗传信息的载体。编码方式的不同决定了遗传算法性能的好坏，也决定了基因型向表现型映射时的解码方法。因此，编码的选择是影响算法性能与效率的重要因素。

编码方法有二进制编码、符号编码、浮点数编码、结构式编码等。

二进制编码是将问题的解空间映射到位串空间上，然后对后者进行操作，得到的结果在通过解码还原成其表现型，以进行适应度的评价。

符号编码方法是用一无数值意义而只有代码含义的符号集来表示个体的每个基因值。

浮点数编码方法是用一个真实值变量的浮点数来表示个体的每个基因值，也就是直接在解的表现型上进行遗传操作。

结构式编码是将问题的解表示成图、树或表等形式。

遗传算子

1.个体选择

从当前种群中选择适应度高和淘汰适应度低的个体的操作过程叫选择。选择操作的主要作用是避免有效基因的损失，其目的是以更大的概率使得优化的个体（或解）生存下来，从而提高计算效益和全局收敛性。

(1)一般的比例选择方法：用正比于个体适配值的概率来选择相应的个体，是一种放回式随机采样的方法。比例选择的基本思想是：个体被选中的概率与它的适应度的大小成正比。设种群的大小为M，群体为p(t)，个体$X_i$ 的适应度大小为$F_i$ ，在一次选择中个体$X_i$被选中概率为p=$Fi$/ $$\sum{i=1}^M F_i$$，在群体中选择M次，即产生新一代群体P(t+1)。

(2)最佳个体保存方法：将群体中适应度最高的一些个体不进行任何交叉操作而直接复制到下一代，用它来替换本代群体中经过遗传操作后适应度最低的个体。此种选择操作有称复制。

2.交叉

遗传算法中的交叉操作的作用是组合出新的个体，方法是将相互配对的染色体按某种方式相互交换其部分基因。遗传算法区别于其他进化算法的重要特征是交叉运算，它在遗传算法中起着关键作用。

交叉操作根据交叉算子将种群中的两个个体以一定的概率随机地在某些基因位进行基因交换，从而产生新的个体。其目的是获得下一代的优良个体，提高遗传算法的搜索能力，对于不同的编码方式，其交叉算子的设计有所不同，常用的交叉算子有：

SSH基础

什么是SSH？

SSH是一种网络协议，用于计算机之间的加密登录。

由于早期互联网通讯都是明文通讯，通讯安全得不到保障。1995年，芬兰学者Tatu Ylonen设计了SSH协议，将登录信息全部加密，成为互联网安全的一个基本解决方案。

##

SSH基本用法

以用户名user登陆远程主机host

如果本地用户名与远程用户名一致，登录时可以省略用户名

SSH默认端口为22，可以通过 -p 修改端口

中间人攻击

用SSH连接远程主机的过程如下：

1.远程主机收到用户的登录请求，把自己的公钥发给用户。

2.用户使用这个公钥，将登录密码加密后，发送回来。

3.远程主机用自己的私钥，解密登录密码，如果密码正确，就同意用户登录。

但是如果有人截获了登录请求，然后冒充远程主机，将伪造的公钥发给用户，获取用户的登录密码，再用这个密码登录远程主机，那么SSH就毫无意义。这就是中间人攻击。

公钥登录

“公钥登录”就是用户将自己的公钥存储在远程主机上。登录的时候，远程主机随机发送来一段字符串，用户用自己的私钥将字符串加密后发送给远程主机，远程主机用之前存储的用户的公钥解密，如果正确，用户就可以直接登录，不在需要密码。

这样，公钥登录就省去了每次登录都需要输入密码的麻烦。
